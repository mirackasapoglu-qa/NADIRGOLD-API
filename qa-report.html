Engeller & bulgular öncelik sırasına göre

BağlantıVerilen adresteki :3001 portu erişilemiyor

api-v2.nadirgold.dev:3001 Cloudflare arkasında ve Cloudflare bu portu proxy'lemiyor → istekler zaman aşımına uğruyor. Çalışan adres portsuz (443): https://api-v2.nadirgold.dev. Test yapılandırması buna göre ayarlandı.

BackendCommerce (sepet) servisi 503 dönüyor

POST /basket/add geçerli token ve geçerli SKU ile bile 503 "Service unavailable" veriyor — servis ayakta değil. basket/get · update · delete ise gateway'de 404. Tüm sepet akışı şu an test edilemiyor 2 test skip

Bot korumasıCloudflare varsayılan Python istemcisini banlıyor (hata 1010)

Python urllib User-Agent'ı 403 ile reddediliyor. Çözüldü: test istemcisine tarayıcı benzeri bir User-Agent header'ı eklendi. CI/otomasyon kuran herkesin bunu bilmesi gerekiyor.

Eksik endpointforgotPasswordSet yolu doğrulanamadı

Dökümandaki /customer/forgotPasswordSet ve denenen tüm varyantlar canlıda 404. İlgili 2 test xfail ile işaretli. İpucu: gateway /customer/forgot-password/* isteğini /auth/forgot-password/* olarak yeniden yazıyor — doğru yol backend'den teyit edilmeli.

Kayıt akışı — teyit edilen kurallar

OtomasyonTest suite kendi hesabını yaratıyor

register OTP kapalı ve doğrudan token dönüyor → dışarıdan test hesabına gerek yok. Kurallar: telefon benzersiz olmalı, TC kimlik no checksum'ı geçerli olmalı (üretici eklendi), başarı yanıtı 200, ve logout Bearer JWT'yi geçersiz kılmıyor (stateless).

Aksiyon maddeleri

  1. Commerce servisini ayağa kaldır — basket/* tümü 503/404. Sepet testleri servis dönünce otomatik koşacak. → Backend / DevOps
  2. Dökümanı düzelt — login, otp/verify, forgot-password yolları + contact 201 status. → Döküman sahibi
  3. forgotPasswordSet doğru yolunu bildir — xfail testler güncellenip aktifleşecek. → Backend
  4. Base URL netleştir — :3001 Cloudflare page-rule ile açılacak mı, yoksa portsuz adres mi standart? → DevOps
  5. Test hesabı için OTP'siz e-posta doğrulaması gözden geçir — şu an kayıt anında emailVerify:1 geliyor. → Backend
